## 目标与边界

- **仓库现状**：后端是 Flask（入口 `run.py`，应用工厂 `app/__init__.py`），MySQL+SQLAlchemy（`app/models.py`），目前无用户/积分/支付/历史记录模块。
- **你要新增**：
  - **用户体系**：微信小程序授权登录（基于 `wx.login` 的 code → 后端换 openid/session）。
  - **积分体系**：改文扣 3 积分（可环境变量配置），筛选/搜索后续可能计费（先做能力但暂不扣费且不记消耗）。
  - **虚拟支付（模拟模式）**：先不接真实微信支付，做“下单→模拟支付成功→到账积分”的链路。
  - **可改写记录**：用户可改写（编辑）改写历史记录、积分消耗记录。
- **小程序端**：位于 `D:/Project/WeChat/ideara/jrtt-tool/`，已有 `wx.login` 但未接后端，HTTP 封装在 `miniprogram/services/transport/http.ts`。

## 关键配置（环境变量）

在后端 `app/config.py` 增加并在 `env.example` 补齐：

- `AUTH_JWT_SECRET`：签发 access token
- `POINTS_REWRITE_COST`：默认 3
- `POINTS_FILTER_COST`：默认 2（先不启用扣费）
- `BILLING_FILTER_ENABLED`：默认 false（先不扣）
- `PAYMENT_MODE`：`mock`（后续可扩展 `wechatpay`）
- `POINTS_PER_YUAN`：默认 10
- `RECHARGE_BONUS_RULES`：默认包含 68→送20、128→送80（可 JSON 配置；30 无赠送）

## 后端接口（新增蓝图 + 改造现有接口）

### A. 鉴权

- 新增 `app/routes/auth.py`
  - `POST /auth/wechat/login`：入参 `{ code }`，后端调用微信 `jscode2session`（需要小程序 `appid/secret` 走环境变量），创建/查找 `users`，签发 JWT（返回 `{ token, user }`）。
- 在 `app/__init__.py` 注册蓝图，并提供 `@require_auth` 装饰器（或 `before_request`）解析 `Authorization: Bearer <token>`。

### B. 积分与扣费

- 新增 `app/routes/points.py`
  - `GET /points/balance`：返回当前积分余额。
  - `GET /points/ledger`：分页返回积分流水。
  - `PUT /points/ledger/{id}`：允许用户编辑 `note`（审计保留 `updated_at`）。
- 改造 `POST /rewrite/start`（在 `app/routes/rewrite.py`）
  - **启用鉴权**：必须登录。
  - **扣费策略**：创建任务前检查余额 ≥ `POINTS_REWRITE_COST`；扣费成功才允许启动。
  - **记账**：写 `points_ledger`（spend/rewrite，delta=-cost）并写 `rewrite_history`。
  - **幂等**：客户端可传 `client_request_id`（可选）避免重复扣费。
- `POST /articles/search`（`app/routes/articles.py`）
  - 先做“计费能力开关”：当 `BILLING_FILTER_ENABLED=false` 时 **不扣费**、也**不写 points_ledger**；但预留字段与分支，未来打开即可。

### C. 模拟充值/支付

- 新增 `app/routes/orders.py`
  - `POST /orders/create`：入参 `{ amount_yuan }`（或 `sku_id`），按规则计算 `points_base=amount*POINTS_PER_YUAN`，加赠送（68/128），生成 `order_no`，返回 `{ order_no, amount_yuan, points_total, pay_mode:'mock' }`。
  - `POST /orders/mock/pay`：入参 `{ order_no }`，将订单置为 paid，并给用户加积分（写 `points_ledger` earn/recharge）。
  - `GET /orders`：订单列表。

### D. 改写历史

- 新增 `app/routes/history.py`
  - `GET /history/rewrite`：分页返回用户的改写历史（从 `rewrite_history`）。
  - `PUT /history/rewrite/{id}`：允许用户编辑 `user_note`、可选编辑展示标题（若你希望允许）。
